# domainc — Per-Program Domain Compiler

## What It Does

domainc reads askic's rkyv output (the parse tree) and generates
a Rust crate containing the program's domain types — the
enum-as-index architecture.

```
askic rkyv output → domainc → Rust crate (enums/structs with rkyv derives)
```

One input (rkyv). One output (Rust crate). The generated crate
IS the sema — enum discriminants as bytes.


## Why It Exists

askic's output has strings: `EnumDef { name: "Element", ... }`.
True sema has no strings — `Element` is a real Rust enum with
`Fire`, `Earth`, `Air`, `Water` as discriminant bytes.

domainc bridges this gap. It reads the structured domain
definitions from the parse tree and generates Rust types.
No text intermediary — no .aski round-trip through corec.


## What It Reads

`Vec<RootChild>` from askic's rkyv output. It extracts:

- **EnumDef** → Rust enums (bare, data-carrying, struct variants, nested)
- **StructDef** → Rust structs (typed fields, self-typed fields, nested)
- **NewtypeDef** → Rust tuple structs
- **ConstDef** → Rust consts
- **ModuleDef** → module scope structure (exports, imports)
- **TraitDeclDef** → trait index enum (which traits exist)

It ignores: TraitImpl (has expressions → semac), Process
(has expressions → semac), FFI (semac generates extern blocks).


## What It Generates

For a module like:
```aski
(Elements Element Quality describe)
(Element Fire Earth Air Water)
(Quality Passionate Grounded Intellectual Intuitive)
{Point (Horizontal F64) (Vertical F64)}
Counter U32
{| MaxSigns U32 12 |}
(describe [(describe :@Self Quality)])
```

domainc generates:

```rust
// Scope index enums
pub enum ElementsEnums { Element, Quality }
pub enum ElementsStructs { Point }
pub enum ElementsNewtypes { Counter }
pub enum ElementsTraits { Describe }
pub enum ElementsConsts { MaxSigns }

// Domain enums
pub enum Element { Fire, Earth, Air, Water }
pub enum Quality { Passionate, Grounded, Intellectual, Intuitive }

// Domain structs
pub struct Point { pub horizontal: f64, pub vertical: f64 }

// Newtypes
pub struct Counter(pub u32);

// Consts
pub const MAX_SIGNS: u32 = 12;
```

All types have rkyv derives. All enums with no data get
Copy + Eq + Hash.


## What It Does NOT Generate

- Trait definitions (semac — needs method body compilation)
- Trait implementations (semac — expression compilation)
- Process bodies (semac — expression compilation)
- FFI extern blocks (semac)

domainc generates the NOUNS (types). semac generates the
VERBS (implementations).


## Relationship to corec

corec already has the codegen logic: emit_enum, emit_struct,
emit_derives, map_primitive, needs_omit_bounds. domainc
reuses this as a library.

corec becomes a library + binary crate:
- `corec` (library) — exposes Codegen, parse types
- `corec` (binary) — .aski text → Rust (unchanged)
- `domainc` (new binary) — rkyv → corec types → Codegen → Rust

No codegen duplication. corec stays zero-dependency for
bootstrap. domainc adds aski (for rkyv parse tree types)
as a dependency.

```
domainc depends on: corec (library) + aski (rkyv types) + rkyv
corec depends on: nothing (bootstrap)
```


## Output Crate Structure

```
<program>-domains/
  Cargo.toml       [package] name = "<program>-domains"
  src/
    lib.rs          mod declarations + re-exports
    elements.rs     per-module generated types
```

semac depends on this crate via flake-crates/:
```toml
[dependencies]
astro-domains = { path = "flake-crates/astro-domains" }
```


## Nix Integration

```
askic output → domainc → domain crate (nix derivation)
                              ↓
                         semac (depends on domain crate)
```

Each is a nix derivation. domainc's output is a source tree
that semac's flake populates into flake-crates/.


## Three rkyv Contracts — Same Pattern

```
aski-core    (corec-generated)    — askicc↔askic contract (grammar)
aski         (corec-generated)    — askic↔semac contract (parse tree)
<program>    (domainc-generated)  — semac↔rsc contract (per-program sema)
```

Each contract is a Rust crate with rkyv derives. Each is
generated by a compiler (corec or domainc). Each is shared
between a serializer and a deserializer.


## The Pipeline (Updated)

```
corec        .aski → Rust with rkyv derives (bootstrap tool + library)
aski-core    grammar .aski + corec → Rust rkyv types (askicc↔askic)
aski         parse tree .aski + corec → Rust rkyv types (askic↔semac)
askicc       .synth → rkyv dialect-data-tree (32 dialects)
askic        .aski source → rkyv parse tree (aski types)
domainc      rkyv parse tree → Rust domain crate (per-program sema types)
semac        rkyv parse tree + domain crate → .sema (pure binary, no strings)
rsc          .sema + domain crate → .rs (Rust source)
```

Eight stages. Each with one input and one output.

semac produces SEMA — not Rust. The .sema binary is the domain
tree serialized using the per-program types from domainc. Every
string resolved to a domain variant. Pure binary.

rsc reads .sema and projects it to Rust source. A pure
mechanical projection — each domain variant maps to one Rust
codegen pattern.
